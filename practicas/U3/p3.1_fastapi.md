# **Práctica 3.1**: Desarrollo de una API REST con FastAPI

## Descripción general

En esta práctica el alumnado deberá **crear una API REST completa utilizando FastAPI**, que implemente un **CRUD** (Create, Read, Update, Delete) sobre una colección de ítems.

La temática de los ítems es **libre**: pueden ser películas, recetas, libros, videojuegos, productos, personajes…  
El objetivo es que esta API sirva también como **base para futuros proyectos frontend** en la asignatura de DWEC.

---

## Requisitos funcionales

1. La API debe gestionar una colección de ítems mediante operaciones CRUD:

   - Crear un ítem
   - Obtener todos los ítems
   - Obtener un ítem por ID
   - Actualizar un ítem existente
   - Eliminar un ítem por ID

2. Cada ítem debe tener **al menos 5 campos** de tipos variados, incluyendo:

   - `id` (entero)
   - cuatro o más campos adicionales (por ejemplo: texto, número, booleano, lista, fecha…)

3. La API debe devolver las respuestas en formato **JSON**.

4. Los endpoints deben utilizar los métodos HTTP adecuados:
   - `GET`
   - `POST`
   - `PUT` o `PATCH`
   - `DELETE`

---

## Requisitos técnicos

1. Deben definirse **modelos Pydantic** para:

   - Datos de entrada
   - Datos de salida (si procede)

2. La API debe **validar los datos** recibidos mediante dichos modelos.

3. Los errores deben gestionarse mediante **HTTPException**, incluyendo casos como:

   - Ítem no encontrado
   - Datos inválidos
   - Intento de crear un ítem duplicado (si aplica)

4. Se debe utilizar la estructura mínima recomendada en FastAPI:
   - Definición de modelos
   - Definición de rutas
   - Simulación de un “almacén de datos” (lista en memoria o similar)

---

## Requisitos de calidad

- Código organizado y comprensible.
- Nombres coherentes en variables, funciones y modelos.
- Rutas REST bien diseñadas.
- Respuestas claras y significativas.
- Uso adecuado de los códigos de estado HTTP.

---

## Entrega

Se debe entregar un solo fichero comprimido nombrado `apellidos_nombre_p3.1.zip` que contendrá:

- El fichero principal `main.py` que contiene la API.
- Un fichero `requeriments.txt`con los requerimientos del proyecto.
- Un ejemplo de varias peticiones de prueba tanto satisfactorias como erróneas en un fichero `test_api.rest` efecutable por **Rest Cient**
- Un fichero `README.md` que incluya vuestro nombre, con una breve explicación de cómo se lanza el servidor FastAPI y justificación del dominio de datos elegido.

---

## Objetivo pedagógico

Con esta práctica el alumnado aprenderá a:

- Diseñar y estructurar una API REST real.
- Validar datos con Pydantic.
- Gestionar errores correctamente.
- Implementar operaciones CRUD completas.
- Preparar una API utilizable por proyectos frontend.
